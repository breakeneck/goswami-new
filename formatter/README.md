# Formatter CLI

Консольний інструмент для форматування транскриптів лекцій з використанням LM Studio API.

## Опис

Цей скрипт бере сирі транскрипти з поля `transcribe_txt` та відправляє їх на обробку до локального AI-моделі через LM Studio API. Результат зберігається у полі `draft`.

## Вимоги

- Python 3.10+
- PostgreSQL база даних
- LM Studio запущений з API сервером (за замовчуванням на `http://localhost:1234/v1`)

## Налаштування

1. Скопіюйте файл конфігурації:
```bash
cp .env.example .env
```

2. Відредагуйте `.env` та вкажіть ваші параметри:
```
# Database settings
DB_NAME=your_db_name
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_HOST=localhost
DB_PORT=5432

# Media files root prefix (path to audio files)
MEDIA_ROOT_PREFIX=~/path/to/media

# LM Studio API settings
LM_STUDIO_API_URL=http://localhost:1234/v1
LM_STUDIO_MODEL=your_model_name

# Processing settings
REQUEST_TIMEOUT=300
MAX_RETRIES=3
TEMPERATURE=0.3
```

## Встановлення

Запустіть скрипт налаштування для створення віртуального середовища:

```bash
./setup.sh
```

Це створить папку `venv/` та встановить усі необхідні залежності.

## Швидкий старт

```bash
# 1. Створити віртуальне середовище (перший раз)
./setup.sh

# 2. Переглянути лекції для обробки
./run.sh list --lang=RUS

# 3. Запустити форматування
./run.sh run --lang=RUS
```

## Використання

### Перегляд доступних команд

```bash
./run.sh --help
```

### Список лекцій для обробки

Показує всі лекції, які мають транскрипт і готові до форматування:

```bash
./run.sh list --lang=RUS
```

### Статус обробки

Показує статистику по всіх лекціях та їх статусах:

```bash
./run.sh status --lang=RUS
```

### Запуск форматування

Починає процес форматування всіх доступних лекцій:

```bash
./run.sh run --lang=RUS
```

## Параметри команд

- `--lang` - мова лекцій (за замовчуванням `RUS`). Доступні варіанти:
  - `RUS` - російська мова
  - `ENG` - англійська мова

## Статуси обробки

- `NULL` - очікує на обробку
- `finished_transcribe` - транскрипція завершена, готово до форматування
- `started_formatting` - йде процес форматування
- `finished_formatting` - форматування завершено

## Обробка помилок

У разі помилки при обробці лекції, деталі записуються у файл з назвою `errors_YYYYMMDD_HHMMSS.txt`, де `YYYYMMDD_HHMMSS` - час запуску скрипта.

Формат запису помилки:
```
2024-01-01 12:00:00 - Помилка при обробці лекції ID=123, title='Назва лекції'
Помилка: [опис помилки]
```

## Як це працює

1. Скрипт завантажує з бази даних список лекцій зі статусом `finished_transcribe` або `NULL`
2. Для кожної лекції:
   - Оновлює статус на `started_formatting`
   - Відправляє текст транскрипту до LM Studio API
   - Отримує сформатований текст
   - Зберігає результат у поле `draft`
   - Оновлює статус на `finished_formatting`
3. Після кожної лекції показується прогрес:
   - Кількість оброблених лекцій
   - Відсоток виконання (за кількістю та за тривалістю)
   - Час, що минув з початку
   - Орієнтовний час завершення (ETA)

## Приклад виводу прогресу

```
================================================================================
Прогрес: 5/100 лекцій
Відсоток: 5.0% (за кількістю) / 4.8% (за тривалістю)
Поточна лекція: Бхагавад-гита 7.27
Час обробки: 0:10:30
Залишок часу: ~3:20:00
================================================================================
```

## Примітки

- Переконайтеся, що LM Studio запущений і модель завантажена перед початком обробки
- Для великої кількості лекцій процес може зайняти значний час
- Скрипт автоматично робить паузи між спробами при помилках (exponential backoff)
